# Plantyex Website

Plantyex Website — это учебное fullstack-веб-приложение для обмена растениями между пользователями. Проект объединяет backend на **Node.js + Express**, frontend на **HTML/CSS/JavaScript** и базу данных **PostgreSQL**.

Приложение позволяет регистрировать пользователей, добавлять растения в каталог, просматривать доступные позиции, создавать предложения на обмен и хранить историю обменов.

---

## О проекте

Идея проекта — дать пользователям простую площадку, где можно:

- зарегистрироваться в системе;
- добавить своё растение в общий каталог;
- просматривать список доступных растений;
- видеть владельца растения;
- отправлять предложение на обмен;
- просматривать активные предложения;
- хранить историю обменов в базе данных.

Проект реализован в классической структуре backend-приложения:

- `routes` — маршруты API;
- `controllers` — обработка HTTP-запросов;
- `models` — работа с PostgreSQL;
- `public` — клиентская часть;
- `config` — подключение к базе данных.

## Архитектура проекта

Сервер запускается из файла `server.js` и:

## API

### Пользователи

#### `GET /api/users`
Возвращает список всех пользователей.

#### `POST /api/users`
Создаёт нового пользователя.

Пример тела запроса:

```json
{
  "name": "Anna",
  "email": "anna@example.com",
  "password": "123456"
}
```

---

### Растения

#### `GET /api/plants`
Возвращает список всех растений вместе с данными владельца.

#### `POST /api/plants`
Добавляет новое растение.

Пример тела запроса:

```json
{
  "user_id": 1,
  "name": "Роза",
  "species": "Rosa rubiginosa",
  "description": "Красивые красные цветы",
  "region": "Европа"
}
```

---

### Предложения обмена

#### `GET /api/offers`
Возвращает список предложений обмена.

#### `POST /api/offers`
Создаёт новое предложение обмена.

Пример тела запроса:

```json
{
  "sender_id": 1,
  "receiver_id": 2,
  "plant_id": 7,
  "status": "pending"
}
```

---

### История обменов

#### `GET /api/history`
Возвращает историю обменов.

#### `POST /api/history`
Добавляет новую запись в историю обменов.

Пример тела запроса:

```json
{
  "sender_id": 1,
  "receiver_id": 2,
  "plant_id": 7
}
```

## База данных

В проекте используется PostgreSQL. В репозитории есть файл `dump.sql`, содержащий структуру и данные для следующих таблиц:

- `users`
- `plants`
- `exchange_offers`
- `exchange_history`

Связи между таблицами реализованы через внешние ключи:

- `plants.user_id -> users.id`
- `exchange_offers.sender_id -> users.id`
- `exchange_offers.receiver_id -> users.id`
- `exchange_offers.plant_id -> plants.id`
- `exchange_history.sender_id -> users.id`
- `exchange_history.receiver_id -> users.id`
- `exchange_history.plant_id -> plants.id`

## Запуск проекта локально

### 1. Клонирование репозитория

```bash
git clone https://github.com/Dodosia/PlantyexWebsite.git
cd PlantyexWebsite
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Подготовка базы данных

Создайте базу данных PostgreSQL и импортируйте дамп:

```bash
psql -U <user> -d <database> -f dump.sql
```

### 4. Настройка подключения к БД

Подключение к PostgreSQL сейчас задаётся напрямую в файле `config/db.js`.

Для локального запуска необходимо:

- указать свои параметры подключения;
- либо адаптировать проект под переменные окружения (`.env`).

### 5. Запуск сервера

```bash
node server.js
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:3000
```

## Особенности текущей реализации

Проект хорошо подходит как учебный fullstack-проект и демонстрирует:

- работу Express-приложения;
- организацию кода по слоям;
- подключение PostgreSQL;
- выполнение SQL-запросов из Node.js;
- взаимодействие frontend и backend;
- базовую предметную модель маркетплейса/сервиса обмена.

При этом в текущей версии есть зоны для улучшения:

- подключение к БД хранится прямо в коде, а не в переменных окружения;
- пароли пользователей сохраняются без хеширования;
- отсутствует полноценная авторизация и аутентификация;
- в `package.json` не добавлены удобные команды запуска (`start`, `dev`);
- часть клиентской логики предполагает дальнейшее развитие API (например, подтверждение обмена);
- валидация входных данных пока минимальная.

Перспективы развития проекта:

- добавить `.env` и пакет `dotenv`;

- добавить Docker для упрощения запуска.
